Why migration to Drupal 8 ?

  • Drupal 6 is reaching EOL
  • No security updates from community
  • Site vunlerable to security threats
  • No more support for d6 (ex: bug fixes or updates)

Why not upgrade to Drupal 7

  • d7 Support is available till 2019
  • d8 support is much longer
  • With little extra effort, you can ensure that your site is secured for next 8 - 10 years
  • New features of d8

Update, Upgrade vs Migration

  • Update: Minor Version Changes
  • Upgrade: Major version Changes (7 -> 8 or 6 -> 8)
  • Upgrade: Inplace code upgrade
  • Migrate: Content and Configuration imports
  • Drupal 8: Only supported method is migration

Migration Support

  • d6 to d8: Core, CCK, Link, Email, Phone and ImageCache
  • d7 to d8: Content, users, taxonomy, blocks, menus and filter formats

Migrations not Supported, yet

  • Views
  • Multilingual Content
  • Entity References

Core Migration Modules

  • Migrate
  • Provides API to migrate content and configuration to a drupal 8 site.
  • Migrate Drupal
  • Provides support to migrate Content and Configuration from d6, d7 to d8

Contributed Modules

Drupal Upgrade

  • Provides necessary tools for core upgrade
  • User Interface @ '/upgrade'
  • Drush command 'migrate-upgrade'

Migrate Tools

  • Provides optional drush & UI tools for managing migrations
  • Drush Commands : status, import, rollback, stop, reset-status, messages, field-source

Migrate Plus

  • Extends the core migration framework with additional functionality
  • Provides API for grouping migrations
  • Facility to manipulate incoming source data
  • Code Examples to build custom migrations

Migrate Manifest

  • Provides drush command for template based sql migration from a manifest file

Planning for Migration

  • Take a Backup
  • Ensure consistent database state at source
  • A fresh Drupal 8 installation
  • Access to source db from the new site
  • Access to source site files
  • Public files are accessed through sites address
  • Private files directory should be directly accessible to new d8 site
  • Install contrib migrate_upgrade module
  • If Drush, install Drush 8
  • If migrating d7 private files, configure D8 file_private_path in settings.php

Enable required modules

  • Migrate wont install any new modules in d8
  • Install all modules on d8 for which Content and Configurations has to be migrated
  • Migrations relevant for modules on both source and destination sites are run
  • Note: Modules may not map one to one

Do not configure Drupal 8 Site

  • Migrate process will override configuration
  • Do not have to create content types and fields manually

Available options (2)

  • Migrate upgrade interface - UI, easier & less control
  • Drush - Robust, faster & allows selective migration
  • Drush - requires additional modules
  • Drush - runs from command line

Executing migration with Drush

  • Drush 8 (composer)
  • Migrate Upgrade
  • Migrate Tools (if more than one time upgrade)

Simple Method

  • migrate_upgrade
  • Generate migrations based on migrate drupal templates & configured source site
  • Executes all created migration in dependency order
  • '--configure-only' - creating migrations only
  • Code:: drush migrate-upgrade --legacy-db=user:pass@local/dbname --legacy-root=d6.com --configure-only
  • drush migrate-status - see list of possible migrations, review and run migrations selectively
  • drush migrate-import {migration_name}
  • drush migrate-import --all

Complex Method

  • migrate-manifest (Migrate Manifest module)
  • Manifest file
  • Set of migrations
  • Run groups of migrations in a reproducible manner
  • 'drush configure-list | grep migrate' - gives the complete list of available migrations

Complex Method ...

  • Creating a manifest yaml file
  • List only the ones you need and their dependencies
  • Migrations can be listed in any order, migrate will sort based on dependencies
  • d6 modules for listed migrations are must. Otherwise migrations won't run properly
  • Drush won't skip the missing migrations and tries to run everything
  • drush migrate-manifest --legacy-db-url=user:pass@local/db manifest.yml

Drush Migration Commands

  • migrate_upgrade
    • Provided by drupal upgrade module
    • upgrade from d6/d7 to d8
    • Generates migrate configuration from source sites content & config
    • then runs migration in proper order (dependencies)
  • Options
    • legacy-db-url
    • legacy-db-prefix
    • legacy-db-legacy-root
    • configure-only

Migration Commands ...

  • migrate-status (ms)
  • migrate-import (mi)
    • to process one or more migration process
    • normally used for custom migrations (non-drupal)
    • can be used to run the created migration configurations
    • eg: drush migrate-import migration_id1, id2
    • Import Options: all, group, limit, feedback, idlist, update, force, etc

More Commands ...

  • migrate-rollback (mr)
  • migrate-stop (ms)
  • migrate-reset-status (mrs)
  • migrate-manifest
  • migrate-messages (mmsg)
  • migrate-field-source (mfs)

Known Issues (6 to 8)

  • Story to Story
  • Page to Basic page
  • Fields missing in the edit form
  • D6: Image nodes and files fields do not get migrated
  • URL Aliases : if a language is not enabled in new d8 site, all aliases wont work

Known issues ...(6 to 8)

  • Menu UI: menu_primary_links_source & menu_secondary_links_source are not migrated
  • Profiles categories (d6) are not grouped in d8
  • Profile field allowed values = allowed values + current selected values in d6
  • Date formats : default, short, medium and long
  • Text/Input formats

Known issues ...(6 to 8)

  • Timezones : daylight saving time
  • Views
  • Home page loads and theme looks broken

Known issues ...(7 to 8)

  • Blocked IPs - iid column is not migrated
  • Menu UI variables
  • Views
  • Filters
